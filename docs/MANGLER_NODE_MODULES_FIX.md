# Mangler: node_modules Overlapping Edit Fix

The TypeScript language service returns rename locations across the entire program, including `.d.ts` files inside `node_modules`; when two different symbol renames resolve to the same offset in a third-party file, the mangler throws `OVERLAPPING edit` and the `-min` build fails after several hours of compile time. The fix in `build/lib/mangle/index.ts` (and its compiled `index.js`) skips any rename location whose path contains `/node_modules/` before registering it as an edit — third-party code is never part of the mangled output and must never be touched. If this error resurfaces after a dependency update, check whether a new package's `.d.ts` declarations share names with mangled symbols and confirm this filter is still in place in both files.
